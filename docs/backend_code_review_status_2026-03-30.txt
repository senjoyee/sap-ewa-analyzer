Backend Code Review Status

Scope: backend

Summary
- A backend security and reliability review identified 7 findings.
- Findings 1-4 are now fixed and pushed.
- Findings 5-7 remain pending.

Findings And Status
1) JWT signature verification was disabled (critical)
- Status: Solved
- What was done:
  - Removed verify_signature=False behavior.
  - Added RS256 JWT signature verification against XSUAA token keys.
  - Added key refresh/reload behavior when kid is not found.

2) Scope authorization was computed but not enforced (high)
- Status: Solved
- What was done:
  - Enforced required scope check (<xsappname>.viewer).
  - Return 403 for insufficient scope.

3) ZIP extraction path traversal risk via extractall (high)
- Status: Solved
- What was done:
  - Added path normalization and traversal guard for each archive entry.
  - Rejects ZIPs with entries escaping extraction directory.

4) Unsafe broad blob deletion based on user-supplied prefix (high)
- Status: Solved
- What was done:
  - Stopped trusting client baseName for bulk delete.
  - Derive base name server-side from fileName.
  - Restrict deletion to a known allowlist of derivative artifact names.
  - Reject path traversal characters in file name.

5) CORS configuration allows wildcard origins with credentials (medium)
- Status: Solved
- What was done:
  - Added CORS_ALLOWED_ORIGINS environment variable parser.
  - Replaced allow_origins=["*"] with environment-specific allowlist.
  - Restricted allow_methods and allow_headers to specific values instead of wildcard.
  - Fallback to localhost origins for development if env var not set.

6) Chat endpoint returns HTTP 200 on upstream model failures (medium)
- Status: Solved
- What was done:
  - Changed error handling to raise HTTPException(500) on OpenAI API failures.
  - Replaced return {"response": f"Error: {err_text}", "error": True} pattern.
  - Returns proper 5xx status code with error detail in HTTP response.

7) Blocking Azure Blob SDK calls inside async routes (medium)
- Status: Solved
- What was done:
  - Wrapped get_blob_properties() in list_files() with asyncio.to_thread().
  - Wrapped list_blobs() and delete_blob() in delete_analysis() with asyncio.to_thread().
  - Wrapped storage_service.get_text_content() and get_bytes() in download_file() with asyncio.to_thread().
  - All blocking Azure SDK calls now offload to threadpool instead of blocking event loop.
